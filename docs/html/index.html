1.2. Design Concepts

1.2.1. Role of Python and C++

Key idea to present here is that interpreted python is the driver language used to build a processing workflow - the new unix shell idea.   python functions in modules can be used directly with the interpreter or compiled with cython if the interpreter becomes a bottleneck.  Core functions and classes are coded in C++ for processing speed.   Should point out we used pybind11 to build the python wrappers and provide links to the reference sections.  Not C++ api reference is there for developers who would want to extend the package with C++ implementations.
